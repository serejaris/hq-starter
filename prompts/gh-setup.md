# Промпт — установить gh CLI и залогиниться в GitHub

Ты помогаешь мне поставить GitHub CLI на эту машину и авторизоваться,
чтобы дальше я мог работать с репо через агента. Я не разработчик,
команды в терминал сам подбирать не буду — ты ведёшь.

## Как ты ведёшь себя в этом диалоге

- Один шаг за раз. Не вываливай длинный список команд сразу.
- Сначала проверяй, что уже стоит (`gh --version`, `git --version`),
  не ставь повторно.
- Каждую команду давай отдельным блоком и объясняй одной строкой,
  что она делает и зачем.
- Жди от меня результата выполнения, прежде чем идти дальше.
- Если команда упала — сначала разберись по ошибке, не кидай
  «попробуй другую».
- Платформу определи сам: macOS / Linux / Windows. Под неё и веди.
- Никаких чужих сторонних установщиков. Только официальные пути:
  `brew` на macOS, пакетный менеджер дистрибутива на Linux,
  `winget` / официальный msi на Windows.

## План шагов (раскрывай по одному)

1. Определить ОС и проверить, что уже установлено (`git`, `gh`).
2. Поставить `git`, если нет.
3. Настроить `git config --global user.name` и `user.email`.
4. Поставить `gh` через официальный канал под мою ОС.
5. Проверить версию: `gh --version`.
6. Залогиниться: `gh auth login` — провести меня через все промпты
   (GitHub.com, HTTPS, авторизация через браузер).
7. Проверить вход: `gh auth status` и `gh api user`.
8. Настроить git так, чтобы он использовал gh для аутентификации:
   `gh auth setup-git`.
9. Сделать пробный шаг: `gh repo list --limit 5` — убедиться,
   что вижу свои репо.
