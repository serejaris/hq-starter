# HQ Starter

Стартовый шаблон личного «штаба» — маленькой папки с markdown-файлами,
по которым удобно держать в одном месте контекст про себя, текущие дела
и активные проекты.

Это не готовая система — это скелет, который ты заполняешь под себя.

Автор: [@ris_ai](https://t.me/ris_ai) — Telegram-канал.

## Зачем это нужно

Когда контекст про тебя разбросан по голове, чатам и заметкам, любая
сборка «кто я и что у меня сейчас» каждый раз начинается с нуля.
HQ собирает это в одно место и в одном формате:

- **`me.md`** — устойчивый контекст: кто ты, чем занимаешься, какой стек,
  какие продукты/услуги, какая аудитория. Меняется редко.
- **`now.md`** — текущий фокус: что в работе сейчас, ритм недели,
  открытые направления. Меняется часто.
- **`projects/`** — по одному файлу на каждый длинный проект:
  что это, статус, цели, следующие шаги.
- **`AGENTS.md`** — короткая инструкция для тех, кто будет читать этот HQ
  (включая тебя через месяц): в каком порядке открывать файлы.

## Как начать

1. Скопируй эту папку себе под нужным именем.
2. Открой `prompts/hq-bootstrap.md` — там готовый промпт, который
   проведёт тебя через 5 коротких вопросов и заполнит файлы по ответам.
3. Если нужно подключить GitHub для синхронизации — `prompts/gh-setup.md`
   проведёт через установку `gh` CLI и авторизацию.

Можно и без промптов: просто открыть каждый шаблон и заполнить руками.
Все файлы — обычный markdown, никакой магии.

## Принципы

- **Один факт — один файл-владелец.** Не дублируй одно и то же
  в нескольких местах. Если факт меняется — правишь в одном.
- **Дата сверху каждого файла.** Так видно, насколько свежий контекст.
- **Коротко.** HQ — карта, а не склад. Длинные материалы живут в других
  местах, в HQ — указатели на них.
- **Решения отдельно от исполнения.** Что делать сейчас — в `now.md`.
  Кто ты вообще — в `me.md`. Не смешивать.

## Структура папки

```
hq-starter/
├── README.md          ← этот файл
├── AGENTS.md          ← как читать этот HQ
├── me.md              ← устойчивый контекст
├── now.md             ← текущий фокус
├── projects/          ← по файлу на каждый длинный проект
│   └── example.md
└── prompts/           ← готовые промпты для заполнения
    ├── hq-bootstrap.md       ← собрать HQ через 5 вопросов
    ├── gh-setup.md           ← поставить gh CLI и залогиниться в GitHub
    ├── yc-rfs-match.md       ← сматчить HQ с YC Requests for Startups
    └── yc-rfs-prd-landings.md ← из совпадений собрать PRD и лендинги
```
